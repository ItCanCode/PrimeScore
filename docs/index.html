Project Overview

PrimeScore is a real-time sports broadcasting and viewer experience tool that provides live updates, visualizations, and commentary feeds for ongoing games.

Match Viewer – View current score, game clock, possession, and key events (goals, fouls, etc.). Display structured data for fans, commentators, or organizers.

Event Feed – View live timeline of in-game events with timestamps and descriptions.– Animate key actions like substitutions or cards.

Match Setup – Manually create matches and add teams, players, and expected schedule.

Live Input – Manually input match events and score changes if not integrated with a live tools.– Supports pause/resume and timeline edits.

High-Level Architecture

PrimeScore uses a client-server architecture:

  • Frontend (Client): React-based UI for users and administrators of the app
  • Backend (Server): Node.js/Express APIs for game data, events, match creation forms and etc...
  • Database: Firestore for real-time updates.
  • Authentication:Firebase is used for authentication with only two roles , sign in as either admin or user

API List

  • GET /api/match/:id – Fetch match details.
  • GET /api/events/:matchId – Retrieve timeline of events.
  • POST /api/event – Add a new game event.

Developer Setup Guide

git clone https://github.com/your-repo/primescore.git
cd frontend (on one terminal)
cd backend (on another terminal)
npm install
npm run dev

Ensure you have Node.js and npm installed before running the commands above.

Git Workflow Overview

  • Branching Strategy: Use main for stable code, feature branches for new work.
  • Pull Requests: All code changes must be reviewed before merging.
  • Commit Messages: Follow the format type(scope): description.
